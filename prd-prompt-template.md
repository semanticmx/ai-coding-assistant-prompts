# Product Requirements Document (PRD) Prompt Template

## SECTION 1: HIGH-LEVEL OVERVIEW AND CONTEXT

Describe the product in one to three paragraphs, connecting its purpose to the company's mission, vision, and core values. Explain how the product supports one or more strategic initiatives from the Balanced Scorecard. Provide the business context, market drivers, and the competitive advantage the product is expected to achieve. Summarize the target audience, key pain points addressed, and the measurable business outcomes expected.

---

## SECTION 2: REQUIREMENTS AND FUNCTIONALITY

### 2.1 Purpose and Objectives

Summarize the primary objectives and the business problem being solved, highlighting the expected business impact and the alignment with strategic goals.

### 2.2 Functional Requirements

List all core functionalities and behaviors in clear, precise terms. Include data inputs, expected outputs, and edge cases.

### 2.3 Non-Functional Requirements

List performance targets, scalability expectations, security measures, compliance needs, and usability goals for MVP delivery.

### 2.4 User Flows

**Customer User Flow:**  
Detail each interaction step from initial awareness through post-engagement follow-ups.

**Admin User Flow:**  
Outline administrative interactions including user management, content updates, order handling, and reporting.

---

## SECTION 3: DESIGN AND USER EXPERIENCE

### 3.1 Core Components

Describe each primary application component, its purpose, and how it interacts with others.

### 3.2 Administrative Components

Detail tools, dashboards, and settings used by admins for management.

### 3.3 UI and UX Guidelines

Provide guiding principles for interface design, accessibility targets, performance optimization, and responsive design considerations. Include any branding requirements that reflect the company's identity.

### 3.4 Communication Flows and Templates

List each trigger event, the intended audience, and the critical data points in communications. Describe tone, branding elements, and personalization rules.

---

## SECTION 4: PLANNING AND EXECUTION

### 4.1 Authentication and Authorization Mechanisms

Describe the approach for managing identity, access control, and token security.

### 4.2 Technical Stack

Recommend an optimized technology stack including presentation layer, business logic layer, hosting, data storage, authentication, and monitoring.

### 4.3 Architecture Overview

Explain how APIs, events, data storage, and static assets will be managed in the chosen architecture pattern.

### 4.4 Security Requirements

Outline encryption standards, compliance mandates, automated security checks, logging, monitoring, and alerting.

### 4.5 CI/CD Workflow

Describe code integration, automated testing, environment promotion, and release management processes.

### 4.6 Availability and Scalability Strategy

Outline strategies to achieve high availability, low latency, and elastic scaling.

### 4.7 Cost Optimization

Detail cost management approaches, including efficient architecture choices, resource optimization, and monitoring tools.

---

## SECTION 5: SUPPORTING INFORMATION

### 5.1 Constraints and Scope

Clearly define MVP boundaries, excluding non-essential features.

### 5.2 Risks and Mitigation Strategies

List potential risks (technical, market, operational) and planned mitigations.

### 5.3 Assumptions and Dependencies

Note any dependencies on other teams, vendors, or technologies.

### 5.4 Success Metrics

List quantitative and qualitative measures to assess success post-launch.
